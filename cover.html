Product Backlogs

Epic: 供应商平台 & 资源管理后台

 Features: 创建邀请账号

 Features: 分类资源管理

 User Story I:


作为超级管理员，我想要为供应商创建账号（账号创建字段：供应商邮箱 / 资源类型 / 用户群组 / 用户权限），

以便供应商可以：收到开通账号邮件 -> 完善账号信息 -> 等待后台审核 -> 接收账号激活邮件 -> 登录账号。



 Features: 素材审核

 Features: 查看数据统计

 User Story II:


作为媒体供应商，我想要收到超级管理员为我开通账号的邮件，完整填写并提交账号信息（企业信息 / 收款账号信息 / 发票信息），

以便超级管理员可以：审核提交的账号信息 -> 通过账号审核，触发系统自动发送的的账号激活邮件。



 User Story III:


作为超级管理员，我想要审核供应商提交的账号信息，

以便供应商可以：

1）审核通过时收到帐号激活邮件 -> 访问激活链接激活账号 -> 登录供应商平台，

2）审核不通过时收到审核不通过邮件 -> 根据审核意见修改账号信息并重新提交审核。



 User Story VII:


作为供应商，我想要在广告位管理中以 API / 批量 / 手工 的方式录入我的户外 / 电视 / 移动广告位信息（可以录入的资源类型为创建账号的超级管理员勾选的资源类型），

以便将这些广告位可以在需求方平台中出售，并获得分成。



 User Story IV:


作为供应商，我想要在成功登录后查看账号信息，修改联系人姓名 / 电话 / 收款账号信息，修改密码，

以便我可以及时更新媒体对接人和收入提现方式。



 User Story V:


作为超级管理员，我想要审核供应商更新的账号信息，

以便供应商的最新账号信息和分成提现方式可以及时生效。



 User Story VI:


作为超级管理员，我想要在管理后台中保存供应商账号信息、审批记录和变更信息，

以便随时查看。



 User Story VIII:


作为供应商，在录入我的移动广告位之前，我想要首先创建移动应用（多个），

以便将我的不同应用中的广告位分开管理，每个广告位获得唯一资源ID。



 User Story IX:


作为超级管理员，我想要审核供应商录入的所有广告位信息，并补充供应商无法提供的某些信息（如多维度位置信息和人群信息），

以便审核通过的广告位可以进入DSP媒体采购库存，在DSP媒介计划可视化工具中准确、全面地展示平台丰富的媒体资源和人群资源。



 User Story X:


作为供应商，我想要根据媒体类型 / 样式类型 / 售卖模式 / 状态 搜索已录入平台的 户外 / 电视 / 移动的广告位条目，

以便我可以在海量资源中快速检索查看需要的广告位信息。



 User Story XI:


作为超级管理员/供应商，我想按照统一的资源ID编号规则登记所有广告位，

以便我可以根据资源ID的编号规则快速识别该条ID的资源类型、供应商 ID、业务种类、售卖模式、价格梯级和媒体类型。



 User Story XII:


作为超级管理员，我想为每个广告位设置分成规则，

以便广告位的交易可以同时为供应商和我方带来预期收益。



 User Story XIII:


作为供应商，我想对已达成采买交易的客户素材进行审核，

以便在我的媒体上出现的广告符合中国法律法规的要求，与我的媒体形象、品牌调性一致，符合我的目标受众的预期。



 User Story XIV:


作为供应商，我想要对客户素材给予审核通过 / 不通过的操作，

以便合规的客户素材可以及时进入媒体曝光，不合格的客户素材可以即时被撤换，保证媒体环境安全和受众体验不被损害。



 User Story XV:


作为供应商，我想在供应商前台首页查看昨天 / 最近3天 / 最近7天的媒体资源变现、使用趋势，我想按 户外 / 电视 / 移动 分别查看不同类型资源的收益概况，

以便我可以随时观察我的媒体资源和销售策略是否有效，帮助我进行资源的优化调整。



 User Story XVI:


作为供应商，我想在流量价值分析中分别查看户外、电视、移动所有已带来收益的广告位，以表格 / 可视化的形式展示这些广告位的收入和曝光趋势，并可以按照媒体 / 样式 / 售卖模式 / 时间自定义筛选对应条件下资源的收益情况，

以便我可以了解资源的变现能力和收入报告。



 User Story XVII:


作为超级管理员，我想在财务结算中查看所有获得分成的供应商的信息和收入情况，包括结算周期 / 结算金额 / 结算币种 / 分成比例 / 业务范围，

以便我可以有效地对供应商的收入进行结算、统计、制表。
